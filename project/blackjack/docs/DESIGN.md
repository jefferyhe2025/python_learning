# Blackjack 开发者设计文档

## 1. 目标与非目标

### 目标

- 用 OOP 表达 21 点：牌、牌堆、手牌、闲家/庄家、对局编排分离清晰。
- 命令行可连续多局游玩，筹码与胜负结算正确。
- 仓库结构接近常见 GitHub 小项目：`README` + `docs` + 分层源码目录。

### 非目标（YAGNI）

- 分牌、保险、加倍
- 多副牌、多闲家、GUI
- 网络对战、持久化排行榜

---

## 2. 整体架构

```
main.py
  └── 组装 Deck / Player / Dealer / ConsoleUI / BlackjackGame
        └── BlackjackGame 编排多局
              ├── 读写 models（状态与规则计算）
              └── 调用 ui（展示与输入）
```

| 层 | 目录 | 职责 | 禁止 |
|----|------|------|------|
| 入口 | `main.py` | 依赖组装、启动 | 堆业务细节 |
| 领域 | `models/` | 牌、堆、手牌点数、筹码 | `input` / 花哨打印 |
| 流程 | `game/` | 下注→发牌→行动→结算→续玩 | 直接拼所有 UI 文案（交给 ui） |
| 交互 | `ui/` | 打印桌面、读入并做格式校验 | 改赔率/改庄家阈值 |

## 4. 类职责与关键行为

下列为**设计名**，实现时可微调命名，但职责勿混层。

### 4.1 `Suite`

- 花色枚举（如 SPADE / HEART / CLUB / DIAMOND）
- 可迭代，供建牌使用

### 4.2 `Card`

- 属性：花色、点数（1–13，A=1，J/Q/K=11/12/13 或等价约定，与展示映射一致即可）
- 行为：可读的字符串表示（如 `♠A`）
- 可选：比较运算仅便于调试/排序，**不作为** 21 点胜负依据

### 4.3 `Deck`

- 持有 52 张 `Card`，维护发牌游标
- `shuffle`：重置游标并乱序
- `deal`：发出下一张；无牌可发时应失败明确（由 Game 保证发牌前牌够或已重洗）
- `has_next`：是否还能发

### 4.4 `Hand`

- 持有当前手牌列表
- `add(card)`：加牌
- `best_value()`：A 在不爆前提下尽量为 11，得到最优点数
- `is_bust()`：点数 &gt; 21
- `is_blackjack()`：恰 2 张且点数为 21
- `clear()`：局间清空

**A 的计算约定**：遍历手牌时先按 A=1 求和，再在不导致爆牌的前提下尽可能把若干张 A 升为 11（通常最多有效升 1 张，因两张 A 升为 11 易爆，算法需正确处理多 A）。

### 4.5 `Player`

- 属性：名字、筹码、`Hand`
- 行为：下注（校验由 UI/Game 先做，对象侧可再断言）、收牌、清空手牌、查询筹码
- 不负责「是否 Hit」的策略（人机由 UI 输入）

### 4.6 `Dealer`

- 独立类；持有 `Hand`
- `should_hit()`：当前最佳点数 &lt; 17 则为 True（**软 17 停牌**）
- 展示区分：对局中一张暗牌，闲家行动结束后翻开

### 4.7 `BlackjackGame`

- 持有：`Deck`、`Player`、`Dealer`、UI、本局 `bet`
- 负责多局循环与单局步骤（见第 5 节）
- 负责结算筹码（见赔率约定）

### 4.8 `ui.console`

- 打印：筹码、手牌、点数、庄家明/暗牌、局结果
- 读取：下注、Hit/Stand、是否续玩
- 非法输入：提示后重试，不污染领域对象

## 5. 一局数据流

多局外层：

```
while 玩家筹码 > 0:
    询问是否退出 → 是则结束会话
    进行一局
会话结束（破产或退出）→ UI 总结
```

单局步骤：

1. **下注**：UI 读入正整数且 ≤ 筹码；扣减 `Player.chips`；保存 `bet`；清空双方 `Hand`。
2. **牌堆**：若剩余牌不足以安全打完一局（建议阈值：剩余 &lt; 20 则整副 `shuffle`），避免中途无牌。
3. **发牌顺序**：闲家、庄家、闲家、庄家（庄家第二张为暗牌）。
4. **自然 Blackjack**：
   - 仅闲家：赔 **3:2**（赢得 `bet * 3/2`，本金已扣则加回 `bet + 赢利`，实现时统一用「结算函数」避免算错）；本局结束。
   - 仅庄家：闲家输（赌注已扣）；本局结束。
   - 双方：Push，退还 `bet`；本局结束。
   - 皆无：进入闲家行动。
5. **闲家行动**：展示己方牌与点数、庄家明牌；Hit 直到 Bust 或 Stand。
6. **若闲家 Bust**：立即判负；庄家不必要牌；局末。
7. **庄家行动**：翻开暗牌；while `should_hit()` 要牌。
8. **比较结算**：
   - 庄家 Bust → 闲家赢 **1:1**
   - 比点数：高赢 1:1，低输，相等 Push 退注
9. **局末**：打印结果与剩余筹码；回到外层循环。

### 结算与筹码（实现核对表）

设本局开始时已扣除 `bet`：

| 结果 | 筹码变化（相对已扣 bet 后） |
|------|-----------------------------|
| 输 | 不再加回 |
| Push | `+= bet` |
| 普通赢 | `+= bet * 2`（退本 + 1:1） |
| 闲家 BJ 赢 | `+= bet + floor/精确的 3:2 赢利`（建议用整数筹码：`bet * 3 // 2` 为纯赢利，再加回本金 `bet`，即 `+= bet + bet * 3 // 2`） |

设计默认：**整数筹码**；下注应为整数，避免浮点赔付。

## 6. 错误处理与边界（冻结）

| 情况 | 策略 |
|------|------|
| 下注非整数 / ≤0 / &gt; 筹码 | UI 拒绝并重试 |
| Hit/Stand、续玩选项非法 | UI 重试 |
| `deal` 时无牌 | 视为编程错误；Game 发牌前保证已洗够 |
| 初始筹码 | 常量，如 `INITIAL_CHIPS = 1000`，README 可说明修改处 |

**规则冻结：**

| 项 | 值 |
|----|-----|
| 软 17 | 庄家停 |
| BJ 赔率 | 3:2 |
| 普通赢 | 1:1 |
| Push | 退本 |
| 闲家 Bust | 庄家不再要 |

## 7. 测试思路

优先测无 I/O 的纯逻辑：

1. **Hand**：典型 A 组合、爆牌、两张 BJ
2. **Deck**：52 张唯一；`shuffle` 后张数不变；`deal` 与 `has_next`
3. **结算**：BJ 3:2、双 BJ push、闲家 bust、庄家 bust、比点输赢
4. **（可选）Game**：注入假 UI + 可控牌序，跑通 1～2 条路径

手工验收：连续多局、破产结束、主动退出、非法输入不崩溃。

## 8. 扩展点（实现后再考虑）

- 软 17 要牌：只改 `Dealer.should_hit`
- 多副牌：扩展 `Deck` 构造
- 加倍/分牌：新开规则模块，避免把分支堆进 `Hand`
- 替换 UI：保留 `BlackjackGame` 对「抽象 UI」的依赖，便于日后非 CLI

## 9. 实现顺序建议

1. `Suite` / `Card` / `Deck`
2. `Hand`（含点数与 BJ/Bust）
3. `Player` / `Dealer`
4. 结算纯函数或 `BlackjackGame` 内私有结算方法
5. `BlackjackGame` 单局 → 多局
6. `console` UI 接线
7. `main.py` 组装
8. 测试与 README「Usage」示例对齐实机输出

## 10. 决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 完整度 | 教学增强版 | 有筹码与 BJ，仍适合练习 OOP |
| 结构 | 领域对象驱动 + 分层目录 | 对齐例一叙事，且像正规小项目 |
| Dealer | 独立类 | 暗牌与固定策略，避免塞进 Player |
| 软 17 | 停牌 | 规则简单、实现与测试量小 |
| 文档先行 | 仅 README + DESIGN | 先冻结边界再写代码 |
